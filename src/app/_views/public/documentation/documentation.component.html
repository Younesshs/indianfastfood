Routing

Naviger dans l'application

Composants du module Public :


Route par défaut ('') : Lorsque l'utilisateur accède à la racine de l'application (par exemple , http://localhost:4200/), il est redirigé vers le chemin 'public/'.

Route 'public' : Lorsque l'utilisateur accède à 'public', le module PublicModule est chargé de manière asynchrone. Cela signifie que les ressources nécessaires pour cette partie de l'application sont chargées uniquement lorsque l'utilisateur accède à cette route.

Route 'auth' : De même, lorsque l'utilisateur accède à 'auth', le module AuthModule est chargé de manière asynchrone.

Route 'order' : Lorsque l'utilisateur accède à 'order', le module OrderModule est chargé de manière asynchrone.

Route 'restaurant' : Lorsque l'utilisateur accède à 'restaurant', le module RestaurantModule est chargé de manière asynchrone. De plus, la route est protégée par un garde (guard) appelé RestaurantAuthGuard, ce qui signifie que l'accès à cette route est limité aux utilisateurs qui satisfont les conditions définies dans ce garde.

Route 'customer' : Lorsque l'utilisateur accède à 'customer', le module CustomerModule est chargé de manière asynchrone. De plus, la route est protégée par un garde appelé CustomerAuthGuard, limitant l'accès aux utilisateurs autorisés.

Route 'administrator' : Lorsque l'utilisateur accède à 'administrator', le module AdministratorModule est chargé de manière asynchrone. La route est protégée par un garde appelé AdministratorAuthGuard, limitant l'accès aux utilisateurs autorisés.

Route '**' (toutes les autres routes inconnues) : Si l'utilisateur tente d'accéder à une route inconnue, il est redirigé vers 'public/e404'. Cela peut être une page d'erreur 404, par exemple, indiquant que la page demandée n'existe pas.
